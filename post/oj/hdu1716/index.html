HDU1716 - 排列2

Task

Ray又对数字的列产生了兴趣： 现有四张卡片，用这四张卡片能排列出很多不同的4位数，要求按从小到大的顺序输出这些4位数。

Input

每组数据占一行，代表四张卡片上的数字（0<=数字<=9），如果四张卡片都是0，则输入结束。

Output

对每组卡片按从小到大的顺序输出所有能由这四张卡片组成的4位数，千位数字相同的在同一行，同一行中每个四位数间用空格分隔。 每组输出数据间空一行，最后一组数据后面没有空行。

Solution

先排序后用next_permutation做组合，麻烦在打印。当最高位一样时，很难判断哪个排列在行尾，即哪个排列是一行中最大的（对每个最高位都排序一下成本又太高），因此很难判断在哪里需要换行。所以转而判断哪个排列是一行中最小的，只需记录最高位改变即可实现。 打印分几种情况： 1、首位是0，不是四位数，不打印； 2、输出在第一行，第一列，只打印数字； 3、输出在任意行，不在第一列，打印一个空格再打印数字； 4、输出在除第一行以外的任意行，第一列，打印一个换行再打印数字； 用变量first_line_first_comb记录是否为第2种情况，每次进入循环排序后初始化为true，打印完第一行第一列后置为false 用变量num_0记录上一次打印的最高位，在第一次打印时初始化这个变量。 另外要注意不同组输出之间要有空行，故用blankline来指定有没有。题目要求只在第一行开始前没有空行，故进入程序时初始化为false，之后一直保持true

#include<cstdio>
#include<algorithm>
using namespace std;
int main(){
    int nums[4];
    bool blank_line=false;
    while(scanf("%d %d %d %d",nums,nums+1,nums+2,nums+3)!=EOF){
        if(nums[0]==0 && nums[1]==0 && nums[2]==0 && nums[3]==0 )
            break;
        if(blank_line)
            printf("\n");
        blank_line=true;
        sort(nums,nums+4);
        bool first_line_first_comb=true;
        int num_0;
        do{
            // if not 4-bit decimal
            if(nums[0]==0)
                continue;
            // first combination in first line
            else if(first_line_first_comb){
                printf("%d%d%d%d",nums[0],nums[1],nums[2],nums[3]);
                first_line_first_comb=false;
                num_0=nums[0];
            }
            // combinations in a line except first combination
            else if(num_0==nums[0])
                printf(" %d%d%d%d",nums[0],nums[1],nums[2],nums[3]);
            // first combination in lines except first line
            else
                printf("\n%d%d%d%d",nums[0],nums[1],nums[2],nums[3]);
            num_0=nums[0];
        }while(next_permutation(nums,nums+4));
        printf("\n");
    }
    return 0;
}
